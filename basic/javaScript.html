# JavaScript

# JavaScript到底是面向对象还是基于对象？

首先说结论，我认为JavaScript是面向对象的，但是其对象和之前所理解的"基于类的对象"不太一样，在ES6之前JavaScript都没有类的概念，但是已经明确了对象在JavaScript中的作用：

TIP

语言和宿主的基础设施由对象来提供，并且ECMAScript程序即是一系列互相通讯的对象集合

对象(Object),通俗理解可以是任何看得见、摸得着甚至是可抽象的任何事物，在语言中，其实也是对其的一种抽象，个人认为，可以描述为，某些或者某类具有一定特征的事物(代码或者概念，这里其实理解过于片面了，我总觉得深入理解可以作为一个哲学概念)

如果是这样的话，JavaScript确实是可以说是一种面向对象的语言，但是其对象的描述方式不同于用类来描述，而是采用了原型的方式。

# JavaScript对象的特征？

对象具有的特征如下：

  1. 对象具有唯一标识性(唯一标识可以看做是其地址值)
  2. 对象具有行为
  3. 对象具有状态

行为和状态在JavaScript对象中的体现是"属性"，对象的属性，大家都是常用的，并且其具有动态性，可以在运行的时候给对象添加属性等

	let obj={};
	obj.o="vera"

其中，JavaScript对象的属性分为数据属性和访问器属性，这就会涉及到常用的API,Obejct.defineproperty和Object.getOwnPropertyDescripter,前者是改变其属性的特征，后者是获取其属性的特征

# 数据属性和访问器属性

下面具体介绍数据属性和访问器属性：

# 一、数据属性

数据属性就是用来描述对象的键值对，如上Obj.o="vera"，其具有的特征值为：

  • value:属性的值
  • writable:是否可写，就是可不可以改变当前属性的值
  • enumerable:是不是可以枚举
  • configurable:该属性是不是可以配置，那就对应了删除操作和后期用Obejct.defineproperty对该属性进行特征值的修改

writable和configurable的区别

writable：false之后，修改该属性的值，不会起作用，但是可以删除，也可以配置该对象 configurable：false之后，可以修改该属性的值，但是不能进行配置了，也不能删除，相当于状态固定了，但是可以改值

需要注意的是，同一个defineproperty操作，会一次性修改该属性的状态，不会因为先配置configurable：false就报错哦❤

# 二、访问器属性

  • getter:读取属性的时候会被调用
  • setter:设置属性的时候会被调用
  • enumerable:同上
  • configrable:同上
Object.defineProperty(obj,"o",{
	set(v){
		console.log("setter")
		}
	get(){
		console.log("getter")
	}
})
