# ES6相关知识点

# var、let 及 const 区别

需要先了解的几个问题如下？

  • 什么是变量提升？
  • 什么是暂时性死区？

针对第一个问题，什么是变量提升，简单说来，就是在变量还没有被声明的时候，我们就可以使用，其可以解决函数相互引用的问题。

一定要注意同名变量和函数的提升问题，如下代码：

console.log(a) // ƒ a() {}
function a() {
    console.log("1111")
}
var a = 1
console.log(a)//1
a()//a is not a function

因此，函数的提升会优于变量，同名变量的赋值也会影响到函数。

var a = 1
let b = 1
const c = 1
console.log(window.b) // undefined
console.log(window.c) // undefined

function test(){
  console.log(a)
  let a
}
test() //Cannot access 'a' before initialization

注意这里的报错信息：Cannot access 'a' before initialization 因为部分同学认为是不存在变量提升的想法是不正确的，这里只是没有初始化，因此会形成一个暂时性死区（暂时性死区可以理解为，不能在声明前就使用变量）。

针对上面这个问题，不同的编译器报错信息不一样，我个人倾向于认为是没有初始化的，这样也比较便于理解。 所以，可以得出区别：

区别：

  • var、let和const声明的变量都存在变量提升，是因为其在生成变量对象的过程(编译)会被初始化为undefined,但是let和const声明变量的时候不会初始化，会存在一个暂时性死区
  • 在全局作用域在，var声明的变量会被挂载在window上，但是let和const声明的变量不会。
  • let 和 const 作用基本一致，但是后者声明的变量不能再次赋值

# 原型实现继承

在JS中用原型来描述类，原型和原型链也是十分重要的概念，使用原型实现继承的方法有很多，常见的有：

  1. 构造函数继承
  2. 原型链继承
  3. 原型式继承
  4. 组合继承
  5. 寄生式继承
  6. 寄生组合继承

实现上面的几种继承如下：

# 构造函数继承：

//构造函数继承
function Father(){
	//父类函数的一些属性
}
Father.prototype.way=function(){
	//父类函数的一些方法
}
function Child(){
	F.constructor.call(this)
	//利用call apply 实现父类函数构造函数的继承
}

# 原型式继承：

其特点是，借助原型，在已有对象的基础上创建新对象，并且不必因此创建自定义类型

function Create(obj){
	let F=functinon(){
	}
	F.prototype=obj; //注意为简单赋值操作
	return new F() //proto 指向 obj
}

那么这个原型式继承和ES5新出的Object.creat()方法有什么区别呢？

`Object.create()`方法与`Create()`的区别

  • Object.create()如果只传入第一个参数的话，两者没有区别，都可以实现原型式继承
  • Object.create()的第二个参数可以为原型对象上的属性（传入的第一个参数）添加描述信息，与Object.defineProperty的第二个参数一样

实例：

let o={
	"huan":"beauty"
}
let res=Object.create(o,{
	"huan":{
		value:"Clever"
	}
})
console.log(res.huan) //Clever

# 寄生式继承:

在原型式继承的继承上，可以为返回的对象添加自己的方法

function CreatAnother(obj){
	let pro=Object.create(obj);
	pro.newWAy=function(){
		console.log("hhh")
	}
	return pro
}

# 寄生组合模式：

function inherit(Father,Child){
	let pro=Object.create(Father);
	pro.constructor=Child;//增强对象，让构造函数指向子类
	Child.prototype=pro;
}
function Father(){
	//父类函数的一些属性
}
Father.prototype.way=function(){
	//父类函数的一些方法
}
function Child(){
	Father.call(this) //构造函数属性继承
}
inherit(Father,Child) //改变构造函数的指向，并且继承原型对象的方法

# 函数原型上定义extends通用方法

#

需要实现的功能： 构造函数和原型对象的属性，不仅可以继承父类函数的值，同时可以自己添加值

//这个方法简单直接，遍历构造函数的对象，直接赋值
Function.prototype.extends=function(func,options,pros){
    for(let item in func.Constructor){
        this.Constructor[item]=func.Constructor[item]
    }
	for(let item in func.prototype){
        this.prototype[item]=func.prototype[item]
    }
    for(let item in options){
        this.Constructor[item]=options[item]
    }
    for(let item in pros){
        this.prototype[item]=pros[item]
    }
}

//creat函数可以实现构造函数继承（原型上的方法也可以继承）和为当前子类添加任意的方法
Function.prototype.create=function(options){
    let context=this;
    let child=new context;
    for(let item in options){
        child[item]=options[item]
    }
    return child
}

可以在寄生式继承函数CreatAnother的基础上，传入自己想要添加的对象，进行遍历赋值

function CreatAnotherNew(obj,option){
	let pro=Object.create(obj);
	for(let item in option ){
		pro[item]=option[item]
	}
	return pro
}
